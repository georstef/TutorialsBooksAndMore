Το σημαντικότερο στο UI μιας εφαρμογής είναι η συνέπεια, 
οι φόρμες πρέπει να είναι παρόμοιες ώστε να μην ψάχνει ο χρήστης
που πρέπει να γράψει και τι πρέπει να πατήσει κάθε φορά που ανοίγει 
μια φόρμα.

Στο συγκεκριμένο πρόβλημα (modal φόρμα με επιπλέον δεδομένα ενός πίνακα)
αποστέλω 3 εικόνες πρωτότυπων φορμών που δημιουργήθηκαν με το balsamiq [http://webdemo.balsamiq.com/]
και ένα παράδειγμα σε XAML (Microsoft Visual C# 2010 Express)
Με λίγη εκπαίδευση θα μπορούσα να δουλέψω και το Blend αλλά για την ώρα δεν το κατέχω.

Σε τέτοιες βοηθητικές φόρμες συνηθίζω να βάζω σε εμφανές σημείο κάποια στοιχεία 
του κύριου πίνακα (πχ. εδώ έβαλα ΑΦΜ και Όνομα) ενώ αν ήταν για πληρωμή παραγγελίας 
θα είχε τα στοιχεία της παραγγελίας και του πελάτη.

Για τα data προσπαθώ να τα ομαδοποιήσω ώστε να γίνει πιο εύκολα η καταχώρηση, 
δηλαδή ότι αφορά το Payment πηγαίνει σε ένα group μόνο του ενώ ότι αφορά 
το Submission πηγαίνει σε άλλο group. Τα component φυσικά είναι ανάλογα με 
το type του πεδίου στον πίνακα (datetime πεδίο σε date edit, text σε textbox κτλ).*

Για τα buttons η εμπειρία λέει "όσο λιγότερα τόσο καλύτερα".


*Το Gateway δεν ξέρω τι θα μπορούσε να είναι και το έβαλα combo.
